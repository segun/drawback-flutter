## Store Listing Best Practices

### Description Writing Tips
- **Lead with benefits, not features**: Focus on what users can do
- **Use formatting**: Bullets, emojis, short paragraphs for readability
- **Include keywords naturally**: Drawing, chat, real-time, collaborate
- **Address concerns**: Privacy, ease of use, no ads
- **Clear call-to-action**: "Download now" or "Join today"
- **Update for new features**: Keep description current with app capabilities

### Screenshot Guidelines
- **Show value immediately**: First screenshot should hook users
- **Include text overlays**: Highlight key features on screenshots
- **Show UI in action**: Not just static screens, show features being used
- **Use consistent branding**: Match app's visual style
- **Demonstrate key flows**: Registration, searching, collaboration
- **Avoid clutter**: Clean, focused screenshots
- **High quality**: No blurriness, proper phone frame if using mockups

### Feature Graphic Tips
- **Eye-catching design**: This is your "billboard" on Play Store
- **App name visible**: Include "DrawkcaB" branding
- **Show key feature**: Collaborative drawing visualization
- **Use brand colors**: Consistent with app identity (rose-based palette)
- **No text-heavy**: More visual, less text
- **Test on multiple devices**: Ensure it looks good at various sizes
